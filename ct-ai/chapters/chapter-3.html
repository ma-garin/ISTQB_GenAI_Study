Original English ▸

Machine learning is a subset of artificial intelligence in which systems learn from data to improve their performance on a task without being explicitly programmed. The three primary paradigms are supervised learning, unsupervised learning, and reinforcement learning.

In supervised learning, a model is trained on labeled data — each training example consists of an input and a corresponding correct output (label). The model learns to map inputs to outputs, and is evaluated on unseen data. Common applications include spam detection, image classification, and credit scoring. Key algorithms include linear regression, decision trees, support vector machines (SVM), and random forests.

In unsupervised learning, no labels are provided. The model discovers structure, patterns, or groupings within the data autonomously. Clustering (k-means, hierarchical clustering) groups similar data points, while dimensionality reduction (PCA) compresses the feature space.

In reinforcement learning, an agent learns by interacting with an environment and receiving reward signals. The agent maximizes cumulative reward over time. Applications include game playing, robotics control, and recommendation systems.

Deep learning is a subset of ML using multi-layer neural networks, capable of learning hierarchical representations from raw data such as images and text.

3.1.1 機械学習の主要パラダイム

機械学習（ML）は人工知能の一分野であり、明示的なプログラミングなしにデータから学習してタスクの性能を向上させるシステムです。主要なパラダイムは教師あり学習・教師なし学習・強化学習の3種類です。

教師あり学習（Supervised Learning）

ラベル付きデータ（入力と正解ペア）を用いてモデルを訓練します。未知データへの予測・分類を目的とします。

  • 応用例： スパムメール検出、画像分類、信用スコアリング、医療診断
  • 主なアルゴリズム： 線形回帰、決定木、サポートベクターマシン（SVM）、ランダムフォレスト
  • 評価指標： 精度・適合率・再現率・F1スコア（→ 3.3.1節参照）

教師なし学習（Unsupervised Learning）

ラベルなしデータから構造・パターン・グループを自律的に発見します。

  • クラスタリング： k-means（データを k 個のクラスタに分割）、階層的クラスタリング（デンドログラムで可視化）
  • 次元削減： 主成分分析（PCA）— 高次元データを低次元で表現。可視化・前処理に有用
  • 応用例： 顧客セグメンテーション、異常検知、推薦システム

強化学習（Reinforcement Learning）

エージェントが環境と相互作用し、報酬シグナルを受け取りながら累積報酬を最大化する行動方策を学習します。

  • 構成要素： エージェント、環境、状態（State）、行動（Action）、報酬（Reward）、方策（Policy）
  • 応用例： ゲームプレイ（AlphaGo）、ロボット制御、推薦システム

Original English ▸

The ML workflow is an iterative, multi-phase process. Phases include: (1) Problem Definition — clarify the business problem and define success criteria; (2) Data Preparation — acquire, clean, and preprocess data; (3) Model Selection — choose an appropriate algorithm and architecture; (4) Model Training — fit the model on training data; (5) Model Evaluation — assess performance on validation and test data; (6) Model Deployment — integrate into production systems; (7) Model Monitoring — track performance and detect drift.

The workflow is not strictly linear. Poor evaluation results trigger a return to data preparation or model selection. Detected drift in production triggers retraining. The train/validation/test split ensures that the final evaluation is performed on data the model has never seen.

Original English ▸

Pretrained models are models trained on large datasets (often billions of parameters) for general tasks. They encode substantial world knowledge and can serve as a reusable base for downstream applications, reducing training time and data requirements significantly.

Transfer learning adapts a pretrained model for a specific task. Fine-tuning continues training on a smaller, task-specific dataset, updating some or all model weights. Parameter-Efficient Fine-Tuning (PEFT) methods, such as LoRA (Low-Rank Adaptation), freeze most weights and train only small adapter layers, reducing computational cost.

Retrieval-Augmented Generation (RAG) combines a retrieval system (searching a knowledge base) with a generative model. At inference time, relevant documents are retrieved and concatenated with the prompt, enabling the model to produce factually accurate, up-to-date responses without retraining. RAG reduces hallucination and keeps information current.

AI-3.1.4K2事前学習済みモデル・ファインチューニング・RAGを要約できる

3.1.4 事前学習済みモデル・ファインチューニング・RAG

事前学習済みモデル（Pretrained Model）

大規模データセット（数十億パラメータ規模）で汎用タスクを対象に学習されたモデルです。世界の広汎な知識を符号化しており、下流タスクの再利用可能なベースとして活用できます。GPT・BERT・LLaMA などが代表例です。

転移学習とファインチューニング（Transfer Learning / Fine-Tuning）

事前学習済みモデルを特定タスクに適応させる手法です。タスク固有の小規模データセットで追加訓練し、モデルのウェイト一部または全部を更新します。

  • フルファインチューニング： 全ウェイトを更新。精度は高いが計算コストが大きい
  • PEFT / LoRA（Low-Rank Adaptation）： 大部分のウェイトを固定し、小さなアダプタ層のみを訓練。計算効率が高く、パラメータ数が少ない

RAG（検索拡張生成 — Retrieval-Augmented Generation）

検索システムと生成モデルを組み合わせたアーキテクチャです。推論時に関連文書をナレッジベースから検索し、プロンプトに連結して生成します。

  • 利点： ハルシネーション（幻覚）の削減、知識の最新性の維持、再訓練不要で情報更新が可能
  • 主な用途： 企業内文書の Q&A、カスタマーサポート、医療情報検索

Original English ▸

Data preparation is one of the most time-consuming phases of the ML workflow, often consuming 60–80% of the total project effort. It encompasses data acquisition, preprocessing, feature engineering, and data labeling.

Data acquisition involves collecting data from structured sources (databases, CSV), unstructured sources (text, images, audio), and synthetic data generation (when real data is scarce or privacy-restricted).

Data preprocessing includes: cleaning (handling missing values by imputation or deletion, removing duplicates, treating outliers), transformation (normalization to [0,1] or standardization to zero mean/unit variance, categorical encoding via one-hot or label encoding), and augmentation (for images: rotation, flipping, cropping; for text: paraphrasing, synonym substitution).

Feature engineering selects, transforms, or creates input features that improve model performance. Sampling strategies include stratified sampling (preserves class proportions), random sampling, oversampling (SMOTE for minority classes), and undersampling.

Data labeling and annotation is the process of assigning ground-truth labels to raw data, often performed by human annotators for supervised learning. Annotation quality directly affects model quality.

AI-3.2.1K2データ準備に関する活動を説明できる

3.2.1 データ準備の活動

データ準備は MLワークフロー中最も時間を要するフェーズであり、プロジェクト全工数の60〜80%を占めることがあります。大きく4つの活動から構成されます。

1. データ収集（Data Acquisition）

  • 構造化データ： データベース、CSV、スプレッドシート
  • 非構造化データ： テキスト、画像、音声、動画
  • 合成データ： GAN・シミュレーションによる人工データ生成（実データが希少または個人情報規制がある場合）

2. データ前処理（Data Preprocessing）

  • クリーニング：
    • 欠損値処理 — 平均/中央値補完（imputation）または削除
    • 重複レコードの除去
    • 外れ値の特定と処理（IQR法・Zスコア法）
  • 変換（Transformation）：
    • 正規化（Normalization）— 値を [0, 1] の範囲にスケーリング
    • 標準化（Standardization）— 平均0・分散1に変換（Z スコア）
    • カテゴリ変数のエンコーディング — One-Hot エンコーディング、ラベルエンコーディング
  • 拡張（Augmentation）：
    • 画像：回転、反転、クロッピング、色調変換
    • テキスト：パラフレーズ、同義語置換、逆翻訳
  • サンプリング（Sampling）：
    • 層化サンプリング（Stratified Sampling）— クラス比率を維持
    • オーバーサンプリング — SMOTE（少数クラスの合成サンプル生成）
    • アンダーサンプリング — 多数クラスを削減してバランスを取る

3. 特徴量エンジニアリング（Feature Engineering）

モデル性能を向上させるために、入力特徴量を選択・変換・作成するプロセスです。ドメイン知識に基づいて有効な特徴量を設計します。

  • 特徴量選択： 相関分析、重要度スコアによる冗長特徴の除去
  • 特徴量作成： 既存の特徴量を組み合わせた新特徴量の生成（例：年齢×収入の積）

4. データラベリング・アノテーション

教師あり学習では、生データに正解ラベルを付与する作業が必要です。ラベリング品質がモデル品質に直結するため、アノテーターの訓練・品質管理プロセスが重要です。

Original English ▸

The dataset is split into three non-overlapping partitions to enable unbiased model evaluation. The training dataset is used to fit model parameters through optimization. The validation dataset is used during training to tune hyperparameters (learning rate, depth, regularization) and to detect overfitting through early stopping. The test dataset is held out entirely until final evaluation, providing an unbiased estimate of generalization performance.

Common split ratios are 70/15/15 or 80/10/10. For small datasets, k-fold cross-validation is preferred: the data is split into k folds, and the model is trained k times, each time using a different fold as the validation set.

Data leakage occurs when information from the validation or test set inadvertently flows into training, producing optimistically biased results that do not reflect real-world performance. Common causes include feature scaling on the full dataset before splitting, and time-series data not split chronologically.

AI-3.2.3K2訓練・検証・テストデータセットの用途を対比できる

3.2.3 訓練・検証・テストデータセット

公正なモデル評価のため、データセットは重複のない3つのパーティションに分割します。

データセット用途タイミング典型的な割合
訓練データセット
(Training)		 モデルパラメータの最適化（損失関数の最小化） 訓練フェーズ全体 70〜80%
検証データセット
(Validation)		 ハイパーパラメータの調整・早期停止（Early Stopping）・過学習の検出 訓練中（各エポック後） 10〜15%
テストデータセット
(Test)		 最終的な汎化性能の評価（未知データへの性能推定） 開発完了後、1回のみ使用 10〜15%

交差検証（Cross-Validation）

データ量が少ない場合は k-fold 交差検証が有効です。データを k 個のフォールドに分割し、k 回訓練を繰り返します。毎回異なるフォールドを検証に使用し、k 回の結果を平均します。

データリーケージ（Data Leakage）のリスク

検証・テストセットの情報が訓練に流出する「データリーケージ」は、楽観的に偏った評価結果を引き起こし、実運用で性能が大幅に低下する原因となります。

  • よくある原因1： データ分割前に全データセットでスケーリング（正規化）を行う → 分割後に実施すべき
  • よくある原因2： 時系列データを時刻順に分割しない → 未来のデータで過去を予測する形になる
試験ポイント — 3つのデータセットの役割 「検証データセットはモデル選択（ハイパーパラメータ調整）に使い、テストデータセットは最終評価に使う」という区別は頻出です。テストデータセットを複数回使用すると、それは実質的に検証データセットになってしまいます。
Original English ▸

A confusion matrix summarizes classification model performance by tabulating the counts of actual vs. predicted class labels: True Positive (TP), False Positive (FP), False Negative (FN), True Negative (TN).

From the confusion matrix, key metrics are derived: Accuracy = (TP+TN)/(TP+FP+FN+TN); Precision = TP/(TP+FP); Recall = TP/(TP+FN); F1-Score = 2×(Precision×Recall)/(Precision+Recall); Specificity = TN/(TN+FP).

The ROC (Receiver Operating Characteristic) curve plots True Positive Rate (Recall) vs. False Positive Rate across different classification thresholds. The AUC (Area Under the Curve) summarizes the overall discriminative ability of the model; AUC = 1.0 is perfect, AUC = 0.5 is no better than random.

3.3.1 ML機能的性能指標

分類モデルの性能評価の基礎となる混同行列（Confusion Matrix）と、そこから導出される各種指標を理解します。K3 レベルのため、実際に数値を計算できることが求められます。

各セルの定義

  • TP（真陽性）： 実際に陽性で、陽性と予測 → 正解
  • TN（真陰性）： 実際に陰性で、陰性と予測 → 正解
  • FP（偽陽性）： 実際は陰性なのに、陽性と予測 → 誤り（Type I Error）
  • FN（偽陰性）： 実際は陽性なのに、陰性と予測 → 誤り（Type II Error）

性能指標の公式

精度（Accuracy） = (TP + TN) / (TP + FP + FN + TN)
適合率（Precision） = TP / (TP + FP)　　　← 陽性予測の信頼度
再現率（Recall / Sensitivity） = TP / (TP + FN)　← 実陽性の検出率
F1スコア = 2 × (Precision × Recall) / (Precision + Recall)
特異度（Specificity） = TN / (TN + FP)　← 実陰性の正検出率

計算例：TP=80, FP=20, FN=10, TN=90

合計サンプル数 = 80 + 20 + 10 + 90 = 200

精度（Accuracy） = (80 + 90) / 200 = 170 / 200 = 0.850 (85.0%)
適合率（Precision） = 80 / (80 + 20) = 80 / 100 = 0.800 (80.0%)
再現率（Recall） = 80 / (80 + 10) = 80 / 90 ≈ 0.889 (88.9%)
F1スコア = 2 × (0.800 × 0.889) / (0.800 + 0.889)
= 2 × 0.711 / 1.689 ≈ 0.842
特異度（Specificity） = 90 / (90 + 20) = 90 / 110 ≈ 0.818 (81.8%)

ROC曲線とAUC

ROC曲線（受信者操作特性曲線）は、分類閾値を変化させたときの真陽性率（Recall）と偽陽性率（1 - Specificity）のトレードオフを視覚化したものです。

  • AUC（曲線下面積）= 1.0： 完全なモデル
  • AUC = 0.5： ランダム予測と同等（無意味なモデル）
  • AUC > 0.9： 優れた判別能力（目安）
Precision vs. Recall のトレードオフ
  • 医療診断（がん検出）→ 再現率（Recall）を最優先（見逃しを最小化）
  • スパムフィルタ → 適合率（Precision）を重視（正常メールの誤検出を防ぐ）
  • F1スコアは両者のバランスを取る指標

Original English ▸

A neural network consists of artificial neurons (perceptrons) organized in layers. Each neuron computes a weighted sum of its inputs, adds a bias term, and applies a non-linear activation function to produce an output. Common activation functions include ReLU (Rectified Linear Unit: max(0,x)), sigmoid (outputs between 0 and 1), and softmax (multi-class probability distribution).

A deep neural network (DNN) has an input layer, multiple hidden layers, and an output layer. Forward propagation passes data from input through hidden layers to output. Backpropagation computes the gradient of the loss function with respect to each weight using the chain rule, and gradient descent updates weights to minimize loss.

Key hyperparameters include learning rate (step size for gradient descent), batch size (number of training samples per weight update), and epochs (number of full passes over training data). Overfitting occurs when the model performs well on training data but poorly on unseen data; it is mitigated by dropout, L1/L2 regularization, and early stopping.

Specialized architectures: CNNs (Convolutional Neural Networks) use local filters for image processing; RNNs (Recurrent Neural Networks) process sequential data via feedback connections; Transformers use self-attention mechanisms and dominate NLP and multimodal tasks.

3.4.1 深層ニューラルネットワークの構造と動作

ニューロン（パーセプトロン）の動作

人工ニューロンは、複数の入力値を受け取り、重み付き和にバイアスを加え、活性化関数を通じた出力を生成します。

出力 = activation( Σ(wᵢ × xᵢ) + b )
wᵢ：重み、xᵢ：入力、b：バイアス

順伝播（Forward Propagation）と誤差逆伝播（Backpropagation）

  • 順伝播： データが入力層から隠れ層を経て出力層まで流れ、予測値を生成
  • 損失計算： 予測値と正解ラベルの差を損失関数（MSE・交差エントロピー等）で定量化
  • 誤差逆伝播： 連鎖律（Chain Rule）を用いて各重みに対する損失の勾配を計算
  • 勾配降下法： 勾配の方向に各重みを更新し、損失を最小化

主要なハイパーパラメータ

ハイパーパラメータ説明調整の影響
学習率（Learning Rate）勾配降下のステップサイズ大きすぎると発散、小さすぎると収束が遅い
バッチサイズ（Batch Size）1回の重み更新に使うサンプル数小さいと汎化しやすいが遅い
エポック数（Epochs）訓練データ全体を何回繰り返すか多すぎると過学習のリスク

過学習（Overfitting）と対策

過学習とは、訓練データには高精度でも未知データに対して性能が大幅に低下する現象です。対策：

  • ドロップアウト（Dropout）： 訓練中にランダムにニューロンを無効化
  • L1/L2正則化： 損失関数に重みのペナルティ項を追加
  • 早期停止（Early Stopping）： 検証損失が悪化したら訓練を停止

特化型アーキテクチャ

  • CNN（畳み込みニューラルネットワーク）： 局所フィルタで画像の空間パターンを抽出。画像分類・物体検出
  • RNN（再帰型ニューラルネットワーク）： フィードバック接続で時系列データを処理。LSTM・GRU が実用的
  • Transformer： 自己注意機構（Self-Attention）により長距離依存関係を捉える。GPT・BERT などの基盤

Traditional code coverage metrics (statement, branch, path) are defined on source code control flow and are not directly applicable to neural networks, which lack explicit branching logic and instead perform continuous numeric computations.

Neuron coverage (NC) measures the percentage of neurons activated (output above a threshold) by at least one test input. Higher neuron coverage means more neurons have been "exercised." However, simple neuron coverage is easy to achieve and may not correlate with defect detection.

More advanced measures include: k-multisection neuron coverage (KMNC), which divides each neuron's activation range into k sections and measures how many sections are covered; strong neuron activation coverage (SNAC), which focuses on neurons activated beyond their observed training range; and layer boundary coverage (LBC), which considers interactions between consecutive layer neurons.

These coverage measures help guide test case generation for neural network testing, analogous to how code coverage guides traditional testing.

3.4.3 ニューラルネットワークのカバレッジ測定

従来コードカバレッジの限界

ステートメントカバレッジ・分岐カバレッジ・パスカバレッジなどの従来のカバレッジ指標は、ソースコードの制御フローを基準としています。しかしニューラルネットワークには明示的な分岐ロジックが存在せず、連続数値演算を行うため、これらの指標は直接適用できません。

ニューロンカバレッジ（Neuron Coverage: NC）

最も基本的なNN向けカバレッジ指標です。テスト入力集合によって少なくとも1回活性化（出力値が閾値以上）されたニューロンの割合を測定します。

ニューロンカバレッジ = 活性化されたニューロン数 / 全ニューロン数 × 100%

限界：単純な NC は達成が容易で、欠陥検出能力との相関が低い場合がある

高度なカバレッジ指標

指標概要特徴
k-multisection ニューロンカバレッジ（KMNC） 各ニューロンの訓練時の活性化範囲を k 個のセクションに分割し、テストがどれだけのセクションをカバーするかを測定 ニューロンの「動作範囲の多様性」を評価
強ニューロン活性化カバレッジ（SNAC） 訓練時に観測された活性化範囲を超えて活性化したニューロンに着目 境界ケース・分布外入力のテスト促進
層境界カバレッジ（LBC） 隣接する層のニューロン間の相互作用パターンを測定 層間の複雑な相互依存関係を考慮
