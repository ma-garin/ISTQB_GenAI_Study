Original English

Model testing focuses on the ML model itself. It evaluates whether model behavior is acceptable for intended use, whether the model is robust against unusual or hostile inputs, and whether it continues to perform after deployment.

6.1 MLモデルリスクとテスト観点

モデルテストは、MLワークフローの成果物であるモデルそのものを対象にします。入力データテストが「モデルへ入るデータ」を扱うのに対し、モデルテストは「学習済みモデルが意図した用途に対して許容できる振る舞いをするか」を検証します。

主な観点は、機能性能、堅牢性、公平性、説明可能性、ドリフト、過学習（Overfitting）・学習不足（Underfitting）、運用中の性能劣化です。単一の正解だけで判定できない場合が多いため、統計的評価、しきい値、許容範囲、リスクベースの優先順位付けが重要になります。

試験のポイント モデルテストでは「評価データで性能を見る」だけでなく、敵対的入力、データ分布の変化、運用環境での比較、モデル文書レビューまで含めて考える。
Original English

Model documentation review examines intended use, assumptions, training data, known limitations, evaluation results, ethical considerations, and operational monitoring requirements.

6.2 モデル文書レビュー

MLモデルの文書レビューでは、モデルカード、データシート、実験記録、評価レポート、リスク評価、変更履歴などを確認します。レビューの目的は、モデルがどの前提で作られ、どの用途に適し、どの条件では使うべきでないかを明確にすることです。

レビュー観点確認内容
意図した用途対象ユーザー、対象ドメイン、許容される入力、禁止用途。
データ学習・検証・テストデータの由来、代表性、バイアス、ラベル品質。
評価使用したメトリクス、しきい値、評価データ、信頼区間。
制約既知の弱点、外れ値、敵対的入力、説明可能性の限界。
運用監視項目、ドリフト検出、再学習条件、ロールバック基準。
Original English

Functional performance testing for probabilistic ML systems uses metrics and acceptance thresholds rather than exact expected outputs. Results should be interpreted statistically.

6.3 機能性能テスト

確率的なMLシステムでは、各入力に対して厳密な期待出力を1つだけ定義できない場合があります。そのため、精度、適合率、再現率、F1スコア、ROC-AUC、平均絶対誤差などのメトリクスと合格しきい値で判断します。

評価では、データ分割、クラス不均衡、信頼区間、ランダムシード、データリークの有無を確認します。特に安全関連や高リスク用途では、単純な平均性能だけでは不十分で、重要なサブグループごとの性能確認が必要です。

Original English

Adversarial testing explores how models behave under intentionally crafted inputs. Metamorphic testing checks expected relations between multiple inputs and outputs when an exact oracle is unavailable.

6.4 敵対的テストとメタモルフィックテスト

敵対的テスト（Adversarial Testing）は、モデルを誤動作させる目的で作られた入力や、自然には少ないが現実に起こりうる入力に対する堅牢性を検証します。画像分類での微小な摂動、テキスト分類での言い換え、LLMでのプロンプトインジェクションなどが例です。

メタモルフィックテスト（Metamorphic Testing）は、正解が一意に決めにくい場合に、入力と出力の間に成り立つべき関係を使ってテストします。たとえば、画像の明るさを少し変えても分類結果は変わらない、同義語に置換しても意図分類は変わらない、といった関係です。

Original English

Operational testing includes drift testing, overfitting and underfitting detection, A/B testing, and back-to-back testing between model versions.

6.5 運用時モデルテスト

ドリフト（Drift）は、入力データや目的変数の分布、または両者の関係が時間とともに変化する現象です。運用中のMLシステムでは、学習時の性能が維持されているとは限らないため、データドリフト・コンセプトドリフト・性能ドリフトを監視します。

過学習（Overfitting）は学習データに過度に適合して未知データで性能が落ちる状態、学習不足（Underfitting）はモデルが十分なパターンを学習できていない状態です。学習曲線、検証性能、テスト性能、エラー分析を組み合わせて判断します。

A/Bテストは、現行モデルと候補モデルを実運用トラフィックで比較する方法です。バックツーバックテストは、同じ入力セットを複数モデルに与え、出力差分と性能差を比較します。どちらもモデル更新時の回帰リスクを下げるために有効です。
